Hexo Deploy，Forgot the SSH Password,reset it

today i want to post some article to my blog , but i forgot my password .
so i reset my github password , but it is still fail to deploy the post.
With some soluation searching online , i had a password to deploy
also , but i cannot remember anything . To fix this problem , i need to reset this
password for SSH.

** Step1:Remove SSH which in your Github Setting
Click the icon of login ， choose the Setting from the menu list ，then choose the SSH
and GPG keys , delete all ssh keys

* Step2:Creat the New SSH kEY

  in your Git Bach ，input the commend: ssh-keygen -t rsa -C "your email address for github"

 the system will display the context like ：

 Generating public/private rsa key pair.
 Enter file in which to save the key (/c/Users/dell/.ssh/id_rsa):

 it ask you where to save  the key ,as normal, we dont change anythong , just save in default
 file, so u just click Enter Key, then it will display another context like below:

 Enter passphrase (empty for no passphrase):
 Enter same passphrase again:

 System ask you to set password to deploy this post to github.after u set the password
then the system will come for another context like :

Your identification has been saved in /c/Users/dell/.ssh/id_rsa.

Your public key has been saved in /c/Users/dell/.ssh/id_rsa.pub.
The key fingerprint is:
65:69:······02:4b emailname@email.com
The key’s randomart image is:
+–[ RSA 2048]—-+
| |
| . o . |
| . o o = o |
| . o * = o |
| E o + o . |
| . o. . . |
| .. |
+—————–+

then it show that you are success to create ssh .

**Step3: Add SSH to github

first you open the local file :id_rsa.pub（like ： c/Users/HP/.ssh/id_rsa.pub），you can edit with text
then you copy all data .
second you loged in the Github ,go to setting -> ssh and GPG KeyS -> add new ssh, create one
ssh, you paste the data to key which you copied in begin

Step4: confirm the success of setting ssh
in your git bash , input the comand : ssh -T git@github.com
u are success to set ssh for hexo deloy ，if it display the context like :
Hi username! You’ve successfully authenticated, but GitHub does not
provide shell access.
