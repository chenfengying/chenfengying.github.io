Type of Java Collection

Types of Collections in Java
The Java Collections Framework is divided into four main interfaces:

List (Ordered Collection with Duplicates)
Set (Unique Elements, No Duplicates)
Queue (FIFO - First In, First Out)
Map (Key-Value Pair Collection)

  1. List (Ordered Collection with Duplicates)
    A List is an ordered collection where elements can be duplicated and accessed using an index.

Common Implementations of List:
ArrayList (Dynamic array, fast for retrieval, slow for insertion/deletion)
LinkedList (Doubly linked list, fast for insertion/deletion, slow for retrieval)
Vector (Similar to ArrayList, but synchronized for thread safety)
Stack (LIFO - Last In, First Out)
Example (ArrayList)
java
Copy
Edit
import java.util.*;

public class ListExample {
public static void main(String[] args) {
List list = new ArrayList<>();
list.add(“Apple”);
list.add(“Banana”);
list.add(“Apple”); // Duplicates are allowed

    System.out.println(list); // Output: [Apple, Banana, Apple]
}

}

  1. Set (Unique Elements, No Duplicates)
    A Set is a collection that does not allow duplicate elements and does not guarantee order.

Common Implementations of Set:
HashSet (Unordered, backed by a hash table)
LinkedHashSet (Maintains insertion order)
TreeSet (Sorted in natural order, backed by a tree)
Example (HashSet)
java
Copy
Edit
import java.util.*;

public class SetExample {
public static void main(String[] args) {
Set set = new HashSet<>();
set.add(“Apple”);
set.add(“Banana”);
set.add(“Apple”); // Duplicate will be ignored

    System.out.println(set); // Output: [Apple, Banana] (Order may vary)
}

}

  1. Queue (FIFO - First In, First Out)
    A Queue follows the FIFO (First-In, First-Out) principle. Used for tasks like job scheduling.

Common Implementations of Queue:
LinkedList (Can function as both List and Queue)
PriorityQueue (Elements are ordered based on priority)
ArrayDeque (Efficient double-ended queue)
Example (PriorityQueue)
java
Copy
Edit
import java.util.*;

public class QueueExample {
public static void main(String[] args) {
Queue queue = new PriorityQueue<>();
queue.add(30);
queue.add(10);
queue.add(20);

    System.out.println(queue.poll()); // Output: 10 (Smallest element first)
}

}

  1. Map (Key-Value Pair Collection)
    A Map is not a part of the Collection interface but is part of the Java Collections Framework. It stores key-value pairs.

Common Implementations of Map:
HashMap (Unordered key-value storage)
LinkedHashMap (Maintains insertion order)
TreeMap (Sorted in natural order)
Hashtable (Thread-safe version of HashMap)
Example (HashMap)
java
Copy
Edit
import java.util.*;

public class MapExample {
public static void main(String[] args) {
Map<String, Integer> map = new HashMap<>();
map.put(“Apple”, 10);
map.put(“Banana”, 20);
map.put(“Apple”, 30); // Overwrites existing value

    System.out.println(map); // Output: {Apple=30, Banana=20} (Order may vary)
}

}

When to Use Which Collection?
Use List (ArrayList, LinkedList, etc.) when you need ordered data and allow duplicates.
Use Set (HashSet, TreeSet, etc.) when you need unique values without duplicates.
Use Queue (PriorityQueue, LinkedList, etc.) when you need to process elements in a FIFO manner.
Use Map (HashMap, TreeMap, etc.) when you need key-value pairs.
